2. Listas Ordenadas y Desordenadas

Las listas en HTML son esenciales para estructurar contenido de forma clara y accesible. Las listas desordenadas (<ul> - *unordered list*) se usan cuando el orden de los elementos no es relevante. Las ordenadas (<ol> - *ordered list*) se usan para secuencias o pasos a seguir. En ambos casos, cada elemento individual es un <li> (*list item*).

Ejemplo de Código:


        -- Lista Desordenada con viñetas 
         <ul>
            <li>Concepto 1: Etiquetas</li>
            <li>Concepto 2: Listas</li>
         </ul>

Quedaría así:
  • Concepto 1: Etiquetas
  • Concepto 2: Listas

Ejemplo de Código:


        -- Lista Ordenada con letras (Introducimos ol type="A")
        <ol type="A">
            <li>Descargar imágenes</li>
            <li>Optimizar el peso</li>
            <li>Insertar con <code>src</code> y <code>alt</code></li>
        </ol>
Quedaría así:
  1. Descargar imágenes
  2. Optimizar el peso
  3. Insertar con src y alt

3. Inserción y Optimización de Imágenes

La etiqueta <img> es una etiqueta **vacía** que muestra contenido visual. Sus atributos clave son: src (ruta del archivo), alt (descripción para accesibilidad), title (texto que aparece al pasar el ratón) y los atributos dimensionales y para dar tamaño usamos **width** y **height**.
Para cumplir con las buenas prácticas, es obligatorio usar imágenes **previamente optimizadas** en la web. . La optimización es crucial; los formatos recomendados son WebP (máxima eficiencia), JPG (fotografía) y PNG (transparencias).

Ejemplo de Código:


        <img 
            src="img_optimizadas/codigohtml-optimz.jpg" 
            alt="HTML" 
            title="Imagen optimizada para rendimiento" 
            width="150" 
            height="150">

4. Tablas

Las tablas se usan para organizar datos en filas y columnas. Para una estructura correcta, usamos los elementos <thead> para el encabezado de las columnas, <tbody> para el cuerpo de los datos y <tfoot> para resúmenes o totales.

El colspan en una celda de tabla (<td> o <th>) permite que esa celda se extienda a través de múltiples columnas, mejorando la presentación de datos.
El rowspan permite que una celda abarque varias filas, útil para agrupar información relacionada verticalmente.

Una tabla correcta debe estructurarse con <thead>, <tbody> y <tfoot>.

Ejemplo de Código:


    <table border="1">
        <caption>Resumen de Datos</caption>
        <thead>
            <tr>
                <th>Artículo</th>
                <th>Cantidad</th>
            </tr>
        </thead>
        <tbody>
            <tr>
                <td>HTML</td>
                <td>5</td>
            </tr>
        </tbody>
        <tfoot>
            <tr>
                <td colspan="2">Fin del reporte</td>
            </tr>
        </tfoot>
    </table>

5. Enlaces: Etiqueta <a>

La etiqueta **<a>** (anchor) es la principal de la web, permite la navegación entre páginas y sitios. El atributo que le sigue es **href** (Hypertext Reference), que define el destino del enlace.

He creado una tabla para representar todo mucho más organizado

Tipos de Enlaces y Atributos Clave

 
Atributos y Tipos de Enlaces
Atributo/Tipo Definición Significado/Valor Importancia
href Define la URL o la ruta del destino. `otra_pagina.html`, `https://google.com` Obligatorio. Define la navegación.
target Especifica dónde abrir el recurso vinculado. `_blank` (nueva pestaña) Controla la experiencia del usuario al salir del sitio.
title Texto de ayuda que aparece al pasar el ratón (tooltip). "Visitar la web de Google" Accesibilidad y usabilidad.
Interno Enlace a otra página dentro del mismo sitio web. `optimizacion.html` Estructura de navegación del proyecto.
Externo Enlace a un sitio web diferente (requiere protocolo `http/https`). `https://www.w3.org/` Referencia a recursos externos.
Anchor/Fragmento Enlace a una sección específica dentro de la misma página (usando `#ID`). `#seccion-texto` Tablas de contenido o navegación rápida.

Ejemplo de Código de Enlaces


    <!-- Enlace interno a una página del proyecto -->
    <a href="formulario.html" title="Ir a la página de Contacto">Ir al Formulario</a>

    <!-- Enlace externo que abre en una nueva pestaña (seguridad y usabilidad) -->
    <a href="https://es.wikipedia.org/" target="_blank" title="Abrir Wikipedia en una nueva pestaña">Wikipedia</a>

    <!-- Enlace a un correo electrónico (mail-to) -->
    <a href="mailto:tunombre@dominio.com">Enviar Correo</a>

    • 6. Formularios (Concepto Básico)

    El elemento <form> es la etiqueta principal para la recopilación de datos de usuario. El atributo clave es action, que especifica dónde se enviarán los datos (por ejemplo, a un servicio como **Formspree**). Los campos se crean principalmente con <input>, especificando su type (text, email, submit, etc.).

    El atributo name es esencial para que la plataforma de destino (como Formspree) sepa qué dato pertenece a qué campo.

    Ejemplo de Código:

    
        <form method="POST" action="[ENDPOINT DE FORMSPREE]">

        <input type="text" id="nombre" name="nombre_usuario" required>
 
        <label for="email">Email:</label>
        <input type="email" id="email" name="_replyto" required>

    7. Comentarios

    Los comentarios son texto invisible para el navegador y el usuario final. Son cruciales para la **documentación del código**, permitiendo a otros desarrolladores (o a ti mismo en el futuro) entender por qué se hizo un cambio o a qué sección pertenece un bloque de código complejo. Los comentarios **no** afectan la velocidad ni el tamaño final de la página cuando esta se carga, ya que el navegador los ignora.

    La sintaxis es simple, utilizando el signo de exclamación para abrir y el doble guion para cerrar.

    Ejemplo de Código:

    
    <!-- Esto es un comentario: no se mostrará en la página -->
    <div>
    <p>Este párrafo es visible.</p>
    </div>
    <!-- FIN de la sección de contenidos principales -->
    Quedaria así:

    Este párrafo es visible.
