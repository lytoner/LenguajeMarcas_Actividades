CONCEPTOS DE HTML

1. Etiquetas de Texto

Las etiquetas de texto en HTML permiten dar formato y estructura al contenido textual. Algunas de las etiquetas más comunes incluyen <strong> para negrita y énfasis semántico, <em> para cursiva e <u> para subrayado.

 
Tabla de etiquetas de texto más utilizadas
Etiqueta Definición Ejemplo Visual Palabras Claves
<b> / <strong> <b>: Resalta en negrita (estilo).
<strong>: Resalta en negrita, dando **énfasis semántico** (más recomendado). 		Hola Resaltar, negrita, semántico
<em> Sirve para poner cursiva (énfasis). Hola Cursiva, énfasis
<u> Sirve para subrayar. Hola Subrayado
<p> Sirve para definir un párrafo de texto. Párrafo de ejemplo Párrafo, bloque
<cite> Sirve para hacer una cita o referencia a un título. Título Cita, referencia, título
<br> Sirve para insertar un salto de línea simple. Línea
Rota		 Salto de línea, vacío

Ejemplo de Código:


<p>Este es un texto con <strong>negrita</strong> y <em>cursiva</em>.</p>

Quedaría así:

Este es un texto con negrita y cursiva.

2. Listas Ordenadas y Desordenadas

Las listas en HTML son esenciales para estructurar contenido de forma clara y accesible. Las listas desordenadas (<ul> - *unordered list*) se usan cuando el orden no es relevante. Las ordenadas (<ol> - *ordered list*) se usan para secuencias o pasos a seguir. Cada elemento es un <li> (*list item*).

Ejemplo de Lista Desordenada (<ul>):


<ul>
    <li>Concepto 1: Etiquetas</li>
    <li>Concepto 2: Listas</li>
</ul>

Quedaría así:

     
  • Concepto 1: Etiquetas
  • Concepto 2: Listas

Ejemplo de Lista Ordenada (<ol> con tipo):


<ol type="A">
    <li>Descargar imágenes</li>
    <li>Optimizar el peso</li>
    <li>Insertar con <code>src</code> y <code>alt</code></li>
</ol>

Quedaría así:

  1. Descargar imágenes
  2. Optimizar el peso
  3. Insertar con src y alt

3. Inserción y Optimización de Imágenes

La etiqueta <img> es una etiqueta **vacía** que muestra contenido visual. Sus atributos clave son: src (ruta del archivo), alt (descripción para accesibilidad), title (texto al pasar el ratón) y los atributos dimensionales **width** y **height**.
Para cumplir con las buenas prácticas, es obligatorio usar imágenes **previamente optimizadas** en la web. La optimización es crucial; los formatos recomendados son WebP (máxima eficiencia), JPG (fotografía) y PNG (transparencias).

Ejemplo de Código:


<img 
    src="https://placehold.co/150x150/f59e0b/ffffff?text=HTML" 
    alt="Código HTML" 
    title="Imagen optimizada para rendimiento" 
    width="150" 
    height="150">

4. Tablas

Las tablas se usan para organizar datos en filas y columnas. Para una estructura correcta y semántica, usamos los elementos <thead> para el encabezado de las columnas, <tbody> para el cuerpo de los datos y <tfoot> para resúmenes o totales. Una tabla correcta debe estructurarse con <thead>, <tbody> y <tfoot>.

El **colspan** permite que una celda se extienda a través de múltiples columnas. El **rowspan** permite que una celda abarque varias filas, útil para agrupar información relacionada verticalmente. [Image of an HTML table showing colspan and rowspan]

Ejemplo de Código:


<table class="tabla-ejemplo">
    <caption>Resumen de Datos</caption>
    <thead>
        <tr>
            <th>Artículo</th>
            <th>Cantidad</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>HTML</td>
            <td>5</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td colspan="2">Fin del reporte</td>
        </tr>
    </tfoot>
</table>

Quedaría así:

Tabla de Ejemplo
Artículo Cantidad
HTML 5
Fin del reporte

5. Enlaces: Etiqueta <a>

La etiqueta **<a>** (anchor) es la principal de la web, permite la navegación entre páginas y sitios. El atributo clave es **href** (Hypertext Reference), que define el destino del enlace.

Tipos de Enlaces y Atributos Clave

 
Atributos y Tipos de Enlaces
Atributo/Tipo Definición Significado/Valor Importancia
href Define la URL o la ruta del destino. `otra_pagina.html`, `https://google.com` Obligatorio. Define la navegación.
target Especifica dónde abrir el recurso vinculado. `_blank` (nueva pestaña) Controla la UX al salir del sitio.
title Texto de ayuda que aparece al pasar el ratón (tooltip). "Visitar la web de Google" Accesibilidad y usabilidad.
Interno Enlace a otra página dentro del mismo sitio web. `optimizacion.html` Estructura de navegación.
Externo Enlace a un sitio web diferente. `https://www.w3.org/` Recursos externos.

Ejemplo de Código de Enlaces


<!-- Enlace interno a una página del proyecto -->
<a href="formulario.html" title="Ir a la página de Contacto" class="text-link">Ir al Formulario</a>

<!-- Enlace externo que abre en una nueva pestaña -->
<a href="https://es.wikipedia.org/" target="_blank" title="Abrir Wikipedia en una nueva pestaña" class="text-link">Wikipedia</a>

<!-- Enlace a un correo electrónico (mail-to) -->
<a href="mailto:manuuubcn98@gmail.com" class="text-link">Enviar Correo</a>

Quedaría así:

Ir al Formulario Wikipedia Enviar Correo

6. Formularios (Concepto Básico)

El elemento <form> es la etiqueta principal para la recopilación de datos de usuario. El atributo clave es action, que especifica dónde se enviarán los datos (por ejemplo, a un servicio como **Formspree**). Los campos se crean principalmente con <input>, especificando su type.

El atributo name es esencial para que la plataforma de destino sepa qué dato pertenece a qué campo.

Ejemplo de Código:


<form method="POST" action="[ENDPOINT DE FORMSPREE]" class="space-y-4">
    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre_usuario" required>

    <label for="email">Email:</label>
    <input type="email" id="email" name="_replyto" required>

    <input type="submit" value="Enviar Datos">
</form>

7. Comentarios

Los comentarios son texto invisible para el navegador y el usuario final. Son cruciales para la **documentación del código**, permitiendo a otros desarrolladores entender por qué se hizo un cambio. La sintaxis es simple, utilizando el signo de exclamación para abrir y el doble guion para cerrar.

Los comentarios **no** afectan la velocidad ni el tamaño final de la página.

Ejemplo de Código:


<!-- Esto es un comentario: no se mostrará en la página -->
<div>
    <p>Este párrafo es visible.</p>
</div>
<!-- FIN de la sección de contenidos principales -->

Quedaría así:

Este párrafo es visible.
